Análisis de servicios de seguridad (X.800 y RFC 4949)

Análisis de escenarios de seguridad utilizando los servicios de la recomendación ITU-T X.800 y la terminología del RFC 4949.

1. Introducción

La seguridad informática requiere marcos conceptuales claros para analizar incidentes y comunicar resultados de forma estructurada. La recomendación ITU-T X.800 define los servicios fundamentales de seguridad, mientras que el RFC 4949 establece la terminología estándar para describir amenazas, vulnerabilidades y ataques.

Ambos documentos se complementan: X.800 permite identificar qué propiedad de seguridad fue comprometida, y el RFC 4949 permite describir cómo ocurrió el incidente y qué tipo de amenaza estuvo involucrada.

2. Servicios de seguridad según X.800

  • Autenticación
  • Control de acceso
  • Confidencialidad
  • Integridad
  • No repudio
  • Disponibilidad

3. Escenarios de seguridad analizados

Escenario 1: Ransomware con exfiltración

X.800: Confidencialidad, Integridad, Disponibilidad

RFC 4949: Multi-stage attack, Data breach

Escenario 2: Base de datos expuesta

X.800: Confidencialidad, Control de acceso

RFC 4949: Misconfiguration, Exposure

Escenario 3: Ataque a la cadena de suministro

X.800: Integridad, Confidencialidad

RFC 4949: Supply chain attack

Escenario 4: Phishing con robo de credenciales

X.800: Autenticación, Control de acceso

RFC 4949: Credential compromise

Escenario 5: Eliminación de respaldos

X.800: Disponibilidad, Integridad

RFC 4949: Data destruction

Escenario 6: Amenaza interna

X.800: Confidencialidad, Control de acceso

RFC 4949: Insider threat

Escenario 7: Alteración de registros

X.800: Integridad, No repudio

RFC 4949: Audit trail compromise

Escenario 8: Fallo operativo

X.800: Disponibilidad

RFC 4949: Operational failure

Escenario 9: Suplantación de identidad

X.800: Autenticación, Confidencialidad

RFC 4949: Masquerade, Phishing

Escenario 10: Ataque destructivo

X.800: Confidencialidad, Integridad, Disponibilidad

RFC 4949: Destructive attack

4. Reflexión técnica

El análisis demuestra que los servicios de seguridad definidos en X.800 siguen siendo válidos para clasificar incidentes reales. Por su parte, el RFC 4949 permite describir estos eventos con terminología precisa y estandarizada.

La combinación de ambos marcos permite documentar incidentes, evaluar impactos y proponer controles adecuados, considerando no solo ataques externos sino también errores humanos y fallas operativas.
