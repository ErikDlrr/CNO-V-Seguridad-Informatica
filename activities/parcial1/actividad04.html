Mecanismos de defensa en red

Implementación de defensa perimetral con iptables: política restrictiva, control de estado (stateful) y reglas específicas para DNS, SMTP y HTTP.

1. Introducción

Un mecanismo básico y efectivo de defensa en red es el filtrado de paquetes. En Linux, iptables permite definir políticas y reglas para controlar qué tráfico se acepta o se bloquea. La estrategia más segura parte de una política por defecto restrictiva (deny-by-default) y después permite únicamente los servicios necesarios.

2. Desarrollo técnico

2.1 Política restrictiva (deny-by-default)

Se establecen políticas por defecto en DROP para reducir la superficie de exposición. Con esto, cualquier tráfico no contemplado explícitamente será bloqueado.

iptables -P INPUT DROP iptables -P OUTPUT DROP iptables -P FORWARD DROP

2.2 Permitir conexiones establecidas (Stateful firewall)

Para que las respuestas de conexiones legítimas funcionen, se permite el tráfico marcado como ESTABLISHED o RELATED. Esto evita romper sesiones válidas y mantiene el control sin abrir “todo”.

iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT iptables -A OUTPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

2.3 Reglas por servicio (DNS, correo y web)

Después de aplicar la política restrictiva, se agregan excepciones para permitir solo el tráfico requerido: DNS saliente, SMTP entrante/saliente del servidor de correo y HTTP entrante al servidor web, además de HTTP saliente de la red local.

# 3) Aceptar tráfico DNS (TCP) saliente de la red local 192.1.2.0/24 iptables -A OUTPUT -p tcp --dport 53 -s 192.1.2.0/24 -m state --state NEW -j ACCEPT # 4) Aceptar correo entrante (SMTP) desde Internet al servidor 192.1.2.10 iptables -A INPUT -p tcp --dport 25 -d 192.1.2.10 -m state --state NEW -j ACCEPT # 5) Permitir correo saliente (SMTP) a Internet desde el servidor 192.1.2.10 iptables -A OUTPUT -p tcp --dport 25 -s 192.1.2.10 -m state --state NEW -j ACCEPT # 6) Aceptar conexiones HTTP desde Internet al servidor web 192.1.2.11 iptables -A INPUT -p tcp --dport 80 -d 192.1.2.11 -m state --state NEW -j ACCEPT # 7) Permitir tráfico HTTP desde la red local a Internet iptables -A OUTPUT -p tcp --dport 80 -s 192.1.2.0/24 -m state --state NEW -j ACCEPT

4. Reflexión técnica

Defensa realista: menos reglas, mejor control

La parte “importante” no es escribir muchas reglas; es que la política por defecto sea restrictiva y que cada excepción tenga un motivo claro. El control por estado (ESTABLISHED/RELATED) permite que el firewall sea usable sin abrir puertas innecesarias.

El riesgo típico es abrir salidas “por comodidad” (por ejemplo OUTPUT ACCEPT). Si OUTPUT queda abierto, malware o exfiltración tendrían salida fácil. Por eso, limitar DNS/HTTP/SMTP a lo necesario mejora la contención y reduce superficie de ataque.
